# Description Formula — Shared

> This file lives in `shared/` and applies to all channels.
> It defines how the `@script-writer` agent generates video descriptions and hashtags.
> Channel-specific elements (CTAs, links, fixed hashtags) are pulled from each channel's `config.json` at runtime.
> Do not edit this file per-channel — channel customization belongs in `config.json`.

---

## Overview

A YouTube description serves two masters: the algorithm and the human.

- **Algorithm:** Indexes keywords, understands video topic, uses for search and suggested placement
- **Human:** Skims to decide if the video is worth watching, navigates via chapters, takes action via CTAs

The agent writes for both simultaneously. The structure below satisfies both.

---

## Inputs Required at Runtime

The agent pulls these before writing:

**From `production-brief.md`:**
- Locked title
- Angle / thesis
- Primary keyword
- Supporting keywords
- Audience language terms

**From `script.md`:**
- Section titles (in order)
- Word count per section (used to calculate chapter timestamps)

**From channel `config.json`:**
- `description.ctaLines[]` — array of CTA lines in preferred order
- `description.links[]` — array of {label, url} objects
- `description.fixedHashtags[]` — exactly 2 fixed channel hashtags

---

## Timestamp Calculation

Chapter markers require approximate timestamps. The agent derives these from script word counts using the following formula:

```
Runtime per section (seconds) = (section word count / total word count) × total runtime in seconds
Cumulative timestamp = sum of all preceding section runtimes
```

Format: `0:00` for under one hour. Always start with `0:00 — [First Section Title]`.

Round to nearest 5 seconds for clean timestamps.

## Description Structure

### Block 1 — Above the Fold (first 2–3 lines)

**Purpose:** Shows in search results and suggested feed previews without clicking "show more." This is the highest-value real estate in the description.

**Rules:**
- First line: natural language expansion of the title — not a copy, an elaboration. Embeds primary keyword naturally within the first 100 characters.
- Second line: one sentence on what the viewer will get from watching. Benefit-forward.
- Third line (optional): a compelling detail or secondary hook that adds intrigue.
- No links, no hashtags, no CTAs above the fold — they waste prime keyword real estate.
- Target: 150–200 characters total across these lines.

**Example structure:**
```
[Primary keyword expanded into a natural sentence that mirrors the title's intent.]
[What the viewer will understand or know by the end of this video.]
[Optional: One specific detail that raises curiosity or signals depth.]
```

---

### Block 2 — Video Summary

**Purpose:** Gives the algorithm more keyword signal and gives the human enough context to commit to watching.

**Rules:**
- 3–5 sentences
- Written in the channel's voice — not corporate, not academic
- Naturally embeds 2–3 supporting keywords without forcing them
- Briefly describes the core argument or narrative arc without revealing the payoff
- Does not repeat the above-the-fold block verbatim — expands on it

---

### Block 3 — Chapter Markers

**Purpose:** Navigation for the viewer, structural signal for the algorithm, and a major driver of watch time retention.

**Rules:**
- One chapter per major script section
- Always begins with `0:00` — intro/hook section
- Format: `0:00 — Section Title` (em dash, one space each side)
- Section titles pulled directly from `script.md` — do not rename or editorialize
- If a section title is too long for a chapter marker (>40 chars), the agent shortens it to a clean 3–5 word version
- Minimum 3 chapters, no maximum

**Format:**
```
CHAPTERS
0:00 — [Hook / Intro title]
[X:XX] — [Section 1 title]
[X:XX] — [Section 2 title]
[X:XX] — [Section 3 title]
[X:XX] — [Bonus title — if applicable]
[X:XX] — [Outro title]
```

---

### Block 4 — CTAs and Links

**Purpose:** Drive secondary actions — subscribe, related content, external destinations.

**Rules:**
- Pulled entirely from `config.json` → `description.ctaLines[]` and `description.links[]`
- Agent outputs them in the order defined in config — do not reorder
- One blank line between each CTA line for readability
- Keep this block brief — 3–5 lines maximum
- No emoji unless defined in config

**Format:**
```
[CTA line 1 from config]
[CTA line 2 from config]

[Link label]: [URL]
[Link label]: [URL]
```

---

### Block 5 — Hashtags

**Purpose:** Categorical signal to YouTube. Appears at the bottom of the description.

**Rules:**
- Always exactly 2 fixed channel hashtags from `config.json` → `description.fixedHashtags[]`
- Always 2–3 topic-specific hashtags generated from the video's primary keyword and supporting keywords
- Total hashtags: never more than 5
- Format: `#hashtag` lowercase, no spaces within a hashtag, all on one line separated by spaces
- Topic-specific hashtags derived from: primary keyword, most specific supporting keyword, and the niche topic term from production brief audience language
- Always the very last element in the description — nothing after hashtags

**Hashtag construction rules:**
- Use terms the audience actually searches — pulled from `production-brief.md` audience language block
- Specific beats generic: `#ancientrome` beats `#history`
- No hashtag over 20 characters — they become unreadable
- No repeating the channel name as a hashtag unless it's one of the two fixed ones

**Format:**
```
#[fixedhashtag1] #[fixedhashtag2] #[topichashtag1] #[topichashtag2] #[topichashtag3]
```

## Full Description Output Format

The agent delivers the complete description in this format, ready to paste into YouTube Studio:

```
DESCRIPTION OUTPUT
Video: [title]

---

[ABOVE THE FOLD]
[Line 1 — keyword-rich expansion]
[Line 2 — viewer benefit]
[Line 3 — optional intrigue detail]

[VIDEO SUMMARY]
[3–5 sentence paragraph]

CHAPTERS
0:00 — [Section title]
[X:XX] — [Section title]
[X:XX] — [Section title]
[X:XX] — [Section title]
[X:XX] — [Section title — bonus, if applicable]

[CTA line 1]

```

## Notes on Shorts Descriptions

For short-form videos, the description is simplified:

- Above the fold: 1–2 lines only
- No chapter markers (shorts have no meaningful timestamp navigation)
- CTA block: 1–2 lines maximum, pointer to long video if Mode A teaser
- Hashtags: same rule — 2 fixed + 2–3 topic-specific, 5 max
- Total description: keep under 200 characters where possible — shorts viewers rarely expand descriptions
